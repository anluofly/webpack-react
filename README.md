# 项目目录架构
* views 目录用于存放项目功能模块的页面，需要根据路由配置情况分割子级目录
* config 目录存放一些配置目录，如：第三方类库引用，路由配置等
* store 目录用于存放项目store相关的文件，包括数据获取的封装等
* components 目录用于存放非义务组件，或者在多个义务间都需要用到的功能组件
